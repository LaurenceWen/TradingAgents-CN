# TradingAgentsCN Pro v2.0.1 更新说明

发布日期：2026-03-11  
版本定位：稳定性增强 + 交易计划能力升级 + 安装与更新流程完善

## 一、版本亮点

1. 交易计划模块完成一轮较大升级：AI评估、规则生成、建议讨论与一键应用形成闭环。
2. 数据同步可靠性明显提升：新增/强化 JIT 同步与数据新鲜度校验，降低"有数据但不新鲜"风险。
3. Windows 安装与自动更新链路完善，2.0.1 发布流程更标准化。
4. 多项影响使用体验的细节修复：持仓、报表导出、资金清零、浮亏显示、规则展示可读性等。

## 二、新增功能

### 1) 交易计划 AI 优化助手

1. 支持围绕评估结果与 AI 继续讨论优化方向。
2. 支持结构化修改建议勾选并应用到现有规则。
3. 支持不讨论直接应用建议（快速路径）。

### 2) 交易计划规则 AI 生成增强

1. 新增选股规则 AI 生成（必须条件/排除条件/加分项）。
2. 新增择时规则 AI 生成（市场环境/入场信号/确认条件）。
3. 新增风险管理规则 AI 生成（止损/止盈/时间止损/逻辑止损）。

### 3) 评估结果展示升级

1. 从"单一综合分"升级为"等级 + 六维评分"展示。
2. 增加风险规则校准信息，便于解释评分变化来源。

### 4) 报告导出能力增强

1. 新增分析报告批量导出（最多 10 个）。
2. 支持 Markdown / Word / PDF 导出组合。

### 5) Windows 自动更新系统

1. 安装版自动更新能力上线，补齐 manifest / upgrade config / 托盘更新链路。

## 三、功能优化

### 1) 交易计划规则编辑体验

1. 风险管理编辑器支持更完整的止损止盈配置（固定比例/技术位/ATR、分批止盈/移动止盈等）。
2. 滑块数值可视化增强，关键参数显示更直观。
3. 择时规则输入结构优化，入场信号与确认条件语义更清晰。

### 2) 规则展示可读性优化

1. 风险管理中的止盈条件由原始对象展示改为可读文本（避免出现 [object Object]）。
2. 分批止盈按档位展开展示（盈利阈值与卖出比例）。

### 3) 数据源与同步策略

1. AKShare 股票列表获取增加多接口回退策略，降低单接口波动影响。
2. 同步任务线程池与优先级策略优化。

## 四、稳定性与修复

### 1) 数据同步与事件循环问题修复

1. 修复 Tushare 同步服务中"事件循环已关闭"相关问题。
2. JIT 同步链路中的 Mongo/线程隔离与优先级读取问题修复。

### 2) 交易计划评估准确性修复

1. 修复择时信号字段映射问题，避免"页面有内容但评估视为缺失"。
2. 已发布系统评估时支持读取生效态（正式版 + 草稿），避免评估旧版本内容。

### 3) 风险规则数据一致性修复

1. 止损/止盈类型改为严格互斥清洗（三选一），避免同时残留多种类型字段。
2. 前端切换类型与后端模型归一化双重兜底。

### 4) 持仓与资金模块修复

1. 修复持仓浮亏显示异常。
2. 修复资金账户清零相关流程问题。
3. 新增持仓添加校验，减少脏数据写入。

### 5) 导出与报告字段修复

1. 修复 PDF/Word 导出中的字段映射、中文标题与格式问题。
2. 修复部分 JSON 转换失败场景的容错与日志诊断能力。

## 五、安全与合规

1. API Key 日志脱敏：多处日志由明文/前缀输出改为统一脱敏输出，降低密钥泄露风险。
2. 依赖与许可治理：移除 pytdx 相关依赖并补充 GPL 审计文档。
3. 文档与配置指引更新：Tushare 获取地址与相关配置文档统一更新。

## 六、兼容性说明

1. 交易计划评估响应结构升级为等级与维度评分主导，相关前端展示已同步升级。
2. 若外部脚本依赖旧字段，请在升级后按新版字段结构完成适配。
3. 建议统一使用项目虚拟环境运行；已补充 xlrd 版本约束，避免旧版依赖导致运行时语法问题。

## 七、升级建议

1. 升级后优先回归交易计划主链路：编辑、评估、应用建议、保存草稿、发布。
2. 校验数据源同步状态，确认 AKShare/Tushare/BaoStock 回退链路符合预期。
3. 抽样验证 Markdown/Word/PDF 导出格式与字段完整性。
