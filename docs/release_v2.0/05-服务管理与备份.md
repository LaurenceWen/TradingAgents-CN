# 服务管理与备份

## 常用脚本

安装目录根目录通常会提供以下脚本：

- `start_all.ps1`：启动全部服务
- `stop_all.ps1`：停止全部服务
- `restart_all.ps1`：重启全部服务
- `start_services_clean.ps1`：按清理模式启动服务

建议优先使用这些脚本管理系统，而不是手动分别启动 MongoDB、Redis、后端和前端。

## 建议的日常操作

### 正常启动

双击启动入口，或在 PowerShell 中执行：

```powershell
.\start_all.ps1
```

### 正常停止

```powershell
.\stop_all.ps1
```

### 重启服务

```powershell
.\restart_all.ps1
```

## 适合重启的场景

以下情况建议重启全部服务：

- 修改了关键系统配置
- 更新安装完成后
- 模型配置变更后调用异常
- 页面长时间无响应但进程仍在运行

## 日志与运行目录

常见目录：

- `logs/`：运行日志
- `data/`：本地数据
- `runtime/`：运行时文件
- `temp/`：临时文件

排查问题时，优先查看 `logs/`。

## 备份建议

建议至少定期备份以下内容：

- `data/`
- `.env`
- 关键导出文件

如果系统已经完成大量配置，建议在升级前先做一份完整备份。

## 升级前备份

升级前建议执行：

1. 停止全部服务
2. 备份 `data/`
3. 备份 `.env`
4. 如有重要报表或导出结果，也一并备份

## 恢复建议

如需恢复旧环境，建议：

1. 停止全部服务
2. 恢复 `data/`
3. 恢复 `.env`
4. 启动服务并验证登录与分析功能

如果是跨版本回退，建议优先在测试环境中验证后再覆盖正式使用目录。
